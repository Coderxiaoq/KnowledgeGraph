# 后续开发说明

这份文档主要给后续接手的人用，重点说明三件事：怎么把数据导入 Neo4j、怎么用 Docker 或不用 Docker 打开后端 Docs 调试，以及各个端口和接口的返回体长什么样。

## 1. Neo4j 导入方式

### 1.1 需要先准备什么

- Neo4j 服务已启动，Bolt 端口默认是 `7687`
- Neo4j Browser 默认是 `7474`
- 本项目的导入脚本在 [grapth_build/src/load_graph.py](../grapth_build/src/load_graph.py)
- 后端连接配置在 [backend/.env](.env) 和 [backend/core/connect.py](core/connect.py)

当前代码里默认连接的是本机 Neo4j：

```text
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=password
```

如果你本地 Neo4j 的密码不是 `password`，请先把脚本里的密码和后端环境变量改成一致，不然导入和后端查询都会连不上。

### 1.2 导入步骤

导入脚本会做这几件事：

1. 读取 [grapth_build/data/knowledge_graph_final.json](../grapth_build/data/knowledge_graph_final.json)
2. 清空 Neo4j 里的旧数据
3. 创建唯一约束和索引
4. 批量导入 `Skill`、`Role`、`Company` 节点
5. 批量导入 `REQUIRES`、`RECRUITS` 关系
6. 打印导入后的统计信息

直接运行：

```bash
cd /home/noteles/work/KnowledgeGraph/grapth_build/src
python load_graph.py
```

如果你想先看导入结果，可以打开 Neo4j Browser：

```text
http://localhost:7474
```

### 1.3 导入后数据结构

导入到 Neo4j 的数据类型主要是两类：

- 节点：`Skill`、`Role`、`Company`
- 关系：`REQUIRES`、`RECRUITS`

节点里会保存原始属性，比如 `name`、`description`、`salary`、`company_id`、`role_id`、`skill_id` 等。关系里也会保存属性，比如 `weight`、`urgency`、`headcount` 等。

## 2. 后端如何打开 Docs 调试

后端是 FastAPI，默认会自动带 Swagger 文档和 ReDoc。

### 2.1 不用 Docker，直接本机启动

适用场景：想本机断点调试、改代码后立刻看效果。

```bash
cd /home/noteles/work/KnowledgeGraph/backend
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

然后打开：

- Swagger Docs: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`
- 健康检查: `http://localhost:8000/health`

### 2.2 用 Docker 启动后端

适用场景：想要和线上环境更接近，或者前后端一起联调。

```bash
cd /home/noteles/work/KnowledgeGraph
docker compose up -d --build
```

如果想让后端代码改完自动热重载，可以用开发配置：

```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d --build
```

Docker 模式下后端一样可以打开 Docs：

- Swagger Docs: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`

### 2.3 直接看接口有没有连通

```bash
curl http://localhost:8000/health
```

返回示例：

```json
{
	"status": "OK"
}
```

## 3. 端口说明

当前项目里常用端口如下：

- `8000`：FastAPI 后端
- `5173`：前端开发端口
- `7687`：Neo4j Bolt 连接端口
- `7474`：Neo4j Browser 页面端口

如果你是通过 Docker 启动，后端默认会通过 `bolt://host.docker.internal:7687` 连接 Neo4j；如果你在本机直接跑后端，则默认连 `bolt://localhost:7687`。

## 4. 接口返回体结构

### 4.1 图谱查询类接口

以下接口都会返回统一包装结构：

- `GET /api/graph/nodes`
- `GET /api/graph/edges`
- `GET /api/graph/search`
- `GET /api/graph/expand/{node_id}`
- `GET /api/graph/expand/2hop/{node_id}`
- `GET /api/graph/category/{label}`
- `GET /api/graph/recommend/2to1`

统一返回格式如下：

```json
{
	"code": 200,
	"msg": "success",
	"data": {
		"nodes": [
			{
				"id": "s1",
				"label": "Skill",
				"properties": {
					"name": "Python"
				}
			}
		],
		"edges": [
			{
				"source": "r1",
				"target": "s1",
				"relation": "REQUIRES",
				"properties": {}
			}
		]
	}
}
```

字段含义：

- `code`：业务状态码，正常时是 `200`
- `msg`：提示信息，正常时是 `success`
- `data.nodes`：节点数组
- `data.edges`：边数组
- `nodes[].id`：节点唯一 ID
- `nodes[].label`：节点类型，常见是 `Skill`、`Role`、`Company`
- `nodes[].properties`：节点的原始属性
- `edges[].source` / `edges[].target`：边两端节点 ID
- `edges[].relation`：关系类型，如 `REQUIRES`、`RECRUITS`
- `edges[].properties`：关系属性

### 4.2 过滤器接口的返回结构

过滤器接口返回的是当前全局筛选状态，结构不是 `code/msg/data`，而是直接返回：

```json
{
	"node_filters": [],
	"edge_filters": []
}
```

相关接口：

- `GET /api/graph/filter`
- `POST /api/graph/filter`
- `POST /api/graph/filter/add`
- `POST /api/graph/filter/remove`
- `POST /api/graph/filter/clear`

单条过滤项结构如下：

```json
{
	"target": "node",
	"field": "name",
	"value": "Python",
	"op": "contains"
}
```

### 4.3 根路径和健康检查

- `GET /` 的返回：

```json
{
	"message": "Career Path Knowledge Graph backend placeholder"
}
```

- `GET /health` 的返回：

```json
{
	"status": "OK"
}
```

## 5. 常用调试顺序

如果你要快速确认整条链路是否正常，建议按这个顺序排查：

1. 先确认 Neo4j 能连上，Bolt 端口是 `7687`
2. 再跑 `python grapth_build/src/load_graph.py` 导入数据
3. 启动后端后访问 `http://localhost:8000/docs`
4. 用 `/health` 确认后端能访问数据库
5. 再调用 `/api/graph/nodes`、`/api/graph/search` 看返回数据

如果接口返回的数据明显变少，优先检查是否设置了全局过滤器，可以先调用：

```bash
curl http://localhost:8000/api/graph/filter
curl -X POST http://localhost:8000/api/graph/filter/clear
```
